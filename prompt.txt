In this repository, create an application called curiosity LM, that is structured similar to websites such as Claude, ChatGPT, and Gemini that have a right side toolbar for chats, but include the following:
- persistent memory allowing contexts of ALL chats to be stored (as a toggleable feature, whether or not to implement persistent memory) using long term and short term memory stored in a database locally. both long term and short term memories implement time decay (such as impelmented in memoriesDB, search up and impelement the paper) storage create a feature allowing for the creation of manual databases of storage of information (e.g. machine learning topics that have been searched for in the past) that have such databases created manually by the user
- have the optionality to use API formats of claude, gemini, openai, or even ollama (each using the api key of the service provider)
Here is the main feature: tree based chats
- Chats are represented as trees with branches allowing for the exploration of subtopics or words, sentences, concepts, contexts within the main chat that the user wants to explore
- Chat "branches" are made to be able to be previewed, with succinct tidbits of further definition or context provided as a pop up, created with lightweight, fast models (much like Google searches currently with AI generated context for information)
- Chat "branches" can be created by hitting the control key and {1,2,3...} that are preset (bound) to prepending statements such as "I want to learn more about" or "I don't understand" or "what are the specifics of" a highlighted piece of text or (as a popup) a custom input piece of text that would be branched off of a highlighted location (or if the location is not highlighted, determine a locus of branching that is found by similarity with the base branch)
- Each chat would have a tree representation (figure) that would have the main branch which has a fixed length but relative locations of branches given by how far down the main branch a branch is created
